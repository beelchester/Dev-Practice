Query (Read)

Use the db.collection.find() method in the MongoDB Shell to query documents in a collection.

ex:
To read all documents in the collection

use sample
db.movies.find()

To select documents which match an equality condition, specify the condition as a <field>:<value> pair

ex:
use sample
db.movies.find( { "title": "Titanic" } )
or
db.movies.find( { title: "Titanic" } )

Use query operators in a query filter document to perform more complex comparisons and evaluations. Query operators in a query filter document have the following form:
{ <field1>: { <operator1>: <value1> }, ... }

ex:
To return all movies from the sample.movies collection which are either rated PG or PG-13:

use sample
db.movies.find( { rated: { $in: [ "PG", "PG-13" ] } } )

$in operator to check rated contains PG or PG-13

greater than equal operator ($gte)

ex:

To return movies which were released in Mexico and have an IMDB rating of at least 7:

use sample
db.movies.find( { countries: "Mexico", "imdb.rating": { $gte: 7 } } )

$or operator

ex:

To return movies from the sample.movies collection which were released in 2010 and either won at least 5 awards or have a genre of Drama:

use sample

db.movies.find( {
year: 2010,
$or: [ { "awards.wins": { $gte: 5 } }, { genres: "Drama" } ]
} )

$gt operator : greater than like db.users.find({age:{$gt: 18}})

it has 2 arguments
db.collection.find(query,projection)

projection stores what fields to show if it is 1 (true) that field will be show, not shown for 0 (false)



Update

  • To update a single document, use db.collection.updateOne().
  • To update multiple documents, use db.collection.updateMany().
  • To replace a document, use db.collection.replaceOne().

use $set operator

ex:

use sample

db.movies.updateOne( { title: "Tag" },
{
$set: {
plot: "One month every year, five highly competitive friends
hit the ground running for a no-holds-barred game of tag"
}
})

the $set operator update the value of the plot field for the movie Tag.

Delete

  • To delete multiple documents, use db.collection.deleteMany().
  • To delete a single document, use db.collection.deleteOne().

To delete all documents from the sample_mflix.movies collection:
use sample_mflix

db.movies.deleteMany({})


To delete all documents from the sample_mflix.movies collection where the title equals "Titanic":
use sample_mflix

db.movies.deleteMany( { title: "Titanic" } )

To delete the first document from the sample_mflix.movies collection where the cast array contains "Brad Pitt":
use sample_mflix

db.movies.deleteOne( { cast: "Brad Pitt" } )


  • Relationships

The two most common ways are through embedded documents and references.
  1. Embedded Documents: This is where a document is stored within another document. For example, you could have a "User" document that has an "Address" embedded within it. This is useful when the related data is small and always accessed together.
  2. References: This is where one document has a field that contains the id of another document. For example, you could have a "Post" document that has a "Author" field that contains the id of a "User" document. This is useful when the related data is large or not always accessed together.

embedded example:

here we are embedding multiple objects

db.products.insert({
_id:3,
name:"Rubber",
price:1.30,
stock:43,
reviews:[{rating:5},{rating:4}]
})


referencing example:

// Create a new User document
db.users.insertOne({
name: "John Doe",
email: "johndoe@example.com",
});

// Create a new Post document with a reference to the User
db.posts.insertOne({
title: "Hello World",
content: "This is my first post",
author: ObjectId("5f7b9a9c8f9e6f44cbc7f04f")
});
